Lock screen · notification
Aether notification stands out
Stronger green border and heavier icon contrast than surrounding notifications. Situational urgency — "6d overdue" in deep red — is a system observation, not an accusation. The user reads the situation before making a decision.
Pre-attentive · situation not failure
Three tap targets, one flow
"↑ Tap for full detail" opens the app to the action detail screen. "Send now" opens the app to the one-tap confirm (this path). "Not now" is a named deferral — logged as signal, Aether re-evaluates. Never labelled dismiss or ignore.
Named deferral · no shame label
"Send now" is weighted — secondary is not
Primary action is darker green at font-weight 700. Secondary "Not now" is recessive. The hierarchy is clear before reading. A user in a time-pressured, distracted state can identify the primary action without parsing both options.
Ford Williams · pre-attentive hierarchy
Confirm screen
This screen is the product, not friction
Per-instance approval is the architectural response to Tier 3 external actions. This screen is not a workaround — it is the correct default. The draft is visible, the recipient is named, and three options are available. The user sees exactly what will happen before it happens.
Tier 3 · reversibility · trust architecture
Draft preview earns the send
The first two lines of the draft appear here, not in the notification. Enough to confirm it's right without reading the full email. "Edit draft" opens the full draft. The user is never asked to send something they haven't seen.
Transparency before action
Lock-screen send is a trust setting, not a default
Users who want to send directly from the lock screen can enable this in trust settings after using the product. Never surfaced at onboarding. The trust arc runs in one direction: the user initiates, the system never proposes escalation.
Trust arc · user-initiated only

Lock screen · notification
Same notification, different tap
The notification is identical to Path B. The difference is which tap target the user hits. Body tap → deep link to action detail. "Send now" → confirm screen (Path B). The notification design supports both paths from one surface.
One notification · two paths
"↑ Tap for full detail" makes the affordance explicit
iOS and Android both make the notification body tappable by convention. The hint makes this explicit for users who want context before deciding. It is not a button — just a label. No extra tap cost.
Convention + explicit hint · zero friction
Transition screen
Routing state, not a loading screen
The app opens to "Opening action #118" rather than a blank screen or the default tab. This confirms the tap landed somewhere specific. Sub-300ms in production — the visual is an arrival affordance, not a wait state.
Purposeful routing · orientation
Only animation on this screen
The transition ring pulse is the sole active animation. Animation constraint: one at a time, never concurrent. The detail screen has no ambient animation — the amber dot is static. The ring resolves before the detail screen appears.
Animation constraint · one at a time
Action detail screen
Amber dot — pending, not done
The Aether brand dot on this screen is amber (held / pending approval), not green (done). The system has not acted yet — it is asking. Colour semantics are load-bearing: green on this screen would be an architectural misread.
Colour semantics · amber = held
"Already done · you didn't ask"
The buffer row shows the Tier 1 action that already happened silently. Without surfacing this, the screen reads as "Aether always asks before acting." The green label establishes the notification as the exception — silent action is the norm.
Tier 1 visible · notification is the exception
Fraunces on the headline
The reason card headline uses Fraunces 700 rather than Lexend. It is the single most load-bearing sentence on the screen — the situation summary. The display serif gives it authority without aggression.
Typography · situation lands with weight
Design decisions
Decision What Why Status
Urgency signal — overdue "6d overdue" in #b83030 at font-weight 500. Appears in notification sub, confirm sub, and reason card red row. Situational urgency is RSD-safe. The signal attributes the gap to the situation, not the user. Without it, the screen reads as if everything is equally calm — wrong for a 6-day overdue invoice 25 minutes before a call. Settled
Primary action weight "Send now" at font-weight 700, background rgba(61,158,114,0.18), colour #2d7a57. Secondary "Not now" is recessive — no background, muted colour. Ford Williams' pre-attentive hierarchy: the user must be able to identify the primary action before reading both options. A time-pressured ADHD user in an executive-function-depleted state cannot parse two equally weighted options reliably. Settled
Send button darkness Primary send buttons use #2d7a57 (dark forest green) rather than the canonical #3d9e72 (mid green). At light mode background (#f2ede6), #3d9e72 is too soft for an action that requires a response. Darkening to #2d7a57 gives the button authority without breaking semantic colour consistency — it is still unambiguously green = action. Settled
Amber dot on detail screen Aether brand dot on action detail header is amber (#c4841a), static — no pulse animation. The system has not acted yet — it is pending approval. Green would read as "done." Amber = held / pending is the correct semantic. Static because the sole animation on this screen is the transition ring; concurrent animation is prohibited. Settled
Reason card headline typeface Fraunces 700 on the reason card headline. All other card content uses Lexend. The headline is the load-bearing sentence — the situation summary the user reads before deciding. Fraunces gives it display-level authority. Using Lexend would visually equalise it with surrounding body content. Settled
Red signal row tint Unsent invoice row: background rgba(184,48,48,0.06), border rgba(184,48,48,0.18). Other rows use neutral alpha. The three signal rows are not equal weight. The unsent invoice is the reason this notification exists. A tinted background makes the primary signal visually distinct before the label is read. Walter's pre-attentive encoding: signal before conscious processing. Settled
Path order Path B (default) shown first. Path A (trust setting) shown second. Default first, exception second. Path B is the experience for all users. Path A is a user-initiated escalation available after establishing trust. Reversing the order would imply symmetry that does not exist architecturally. Settled
Touch targets All interactive elements: min-height 44px. Primary send buttons: min-height 48px. WCAG 2.5.5 and Ford Williams' mobile accessibility guidelines. ADHD users frequently interact in distracted, stressed, or executive-function-depleted states. Fine motor precision degrades under cognitive load. 44px is the floor; primary actions get additional height. Settled
Button chrome typeface Ghost buttons (Edit draft, Not now, secondary actions) use DM Mono. Primary send buttons use Lexend. DM Mono on chrome elements (labels, secondary actions, metadata) creates visual rhythm and separation from reading surfaces. Lexend on primary actions (send) reads as human voice rather than system label — appropriate for a consequential action. Settled
RSD copy throughout "Not now" (not dismiss/ignore). "Nothing leaves without you." "Nothing sends without you." "Opened from notification." Reason card headline states situation, not user pattern. RSD safety framework: every copy element must attribute a pattern to a system observation, not a personal failure. "Not now" is a named deferral. "Nothing leaves without you" is a control assertion. No copy implies the user has failed, avoided, or fallen behind. Settled · non-negotiable
